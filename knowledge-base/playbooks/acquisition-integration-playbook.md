# Acquisition Integration Playbook

How Pawline integrates an acquired clinic, function by function, stage by stage. Co-owned by the COO and the Head of M&A. The bound version is [[acquisition-integration-playbook-book]].

## Core framework

Five stages: **T-30 / T-day / T+30 / T+90 / T+180**. Each stage has clinical, operational, financial, IT, and people milestones. Slipping any single stage propagates; the playbook's discipline is in not letting that happen.

See [[integration-stages]] for the cross-functional framework definition.

## By function

### Clinical (owned by RMD)

- **T-30**: Chart audit baseline; clinical-quality scorecard. SOC orientation scheduled.
- **T-day**: Controlled-substance log handover. DEA registration coordination.
- **T+30**: First chart audit; SOC orientation completed. Anesthesia tier compliance check.
- **T+90**: SOC compliance check #2. Quality-review participation begins.
- **T+180**: Full participation in [[clinical-quality-review]] cycle.

### Operational (owned by ROD + Practice Manager)

- **T-30**: Practice manager + ROD walk-through. Vendor account inventory. Schedule alignment.
- **T-day**: Cutover weekend — banking, payroll, vendor cutover.
- **T+30**: Operating cadence rhythm established. Quick-win operational fixes identified.
- **T+90**: Standard ops cadence ([[2026-04-22-weekly-ops-review]] participation).
- **T+180**: Indistinguishable from same-tier organic clinic.

### Financial (owned by FP&A + ROD)

- **T-30**: Working capital target validated. Banking + payroll cutover plan.
- **T-day**: Wire transfer. Banking cutover completed.
- **T+30**: First NetSuite period close. P&L rollup verified ([[clinic-pl-rollup]]).
- **T+90**: Legacy AR cleanup substantially complete. Collection ratio normalizing.
- **T+180**: Steady-state P&L.

### IT (owned by Director of Application Engineering)

- **T-30**: Cornerstone tenant provisioned (or PIMS migration scheduled).
- **T-day**: Network + IT cutover.
- **T+30**: Data warehouse ingestion verified.
- **T+90**: Ad-hoc IT issues resolved.
- **T+180**: Migration completed if scheduled within 12 months ([[pims-migration-pathway]]).

### People (owned by Director of Talent + Practice Manager)

- **T-30**: Comp matching analysis; equity grants prepared for retained DVMs.
- **T-day**: Staff announcement; equity grants land for eligible.
- **T+30**: First all-hands by Pawline exec.
- **T+90**: 90-day check-ins with retained DVMs. Stay interviews.
- **T+180**: Standard people cadence.

## Communication discipline

- Pre-close: NDA discipline ([[ma-confidentiality-policy]]) — no premature disclosure to staff or clients
- T-day: simultaneous internal-staff + client-facing announcements
- Through T+30: weekly comms from Practice Manager to staff
- T+90: regional brand integration messaging

## Common failure modes

- Slipping the T+90 chart-audit because the RMD is overstretched — solve by booking the audit at T-30
- Vendor-account chase taking longer than expected — solve by pre-T-30 vendor-account inventory
- Staff resistance to procurement centralization — solve by acknowledging the local-vendor relationships explicitly in T+30 communication

## When this playbook fails

If a clinic is materially behind at T+90, escalate to Marcus + James for a recovery conversation. Recoveries cost 2-3x more than on-track integrations. Better to catch slippage early.
